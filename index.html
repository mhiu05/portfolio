Featured Projects

Non-Invasive Health Analysis System

Nov. 2025 — Present

A full-stack biometric measurement platform that extracts heart rate, HRV, blink rate, and BVP signals from facial videos — featuring real-time webcam analysis via WebSocket, offline video processing with an async job queue, and an AI medical chatbot powered by Advanced RAG.

Live analysis interface — real-time HR, Blink Rate, SNR & BVP waveform

System Architecture

Designed with 4 main layers: Computer Vision & rPPG, Signal Processing, AI/NLP (RAG Chatbot), and Web Fullstack — connected through WebSocket, REST API, and a Celery + Redis distributed job queue.

Deep Dive — Key Components

rPPG Engine & Signal Processing

Computer Vision
  • Deployed FactorizePhys (3D-CNN + NMF-based FSAM) via ONNX Runtime — lightweight inference without PyTorch dependency, supporting optional CUDA acceleration.
  • MediaPipe Face Mesh for real-time face detection, bounding box expansion, and crop resizing to model-compatible input.
  • Chunk-wise processing pipeline: SessionState maintains a 180-frame rolling buffer per session for continuous BVP extraction.
  • Signal processing: detrend → Butterworth bandpass → FFT peak detection for heart rate. Age-adaptive bandpass ranges via get_bandpass_by_age().
  • Extracts comprehensive HRV metrics: RMSSD, SDNN, PNN50, plus SNR quality assessment.
0.04 MAE (bpm) — Sitting still
0.83 MAE (bpm) — Head motion
1.67 MAE (bpm) — Talking
<2 bpm error — All conditions

Real-time WebSocket Streaming

Live Analysis
  • WebSocket endpoint (ws://host/ws/stream) accepts live webcam frames from the browser, returning face bounding box and vitals in real-time.
  • Protocol: Client sends {"type":"frame","data":"..."} (base64 JPEG). Server responds with face / vitals / error messages.
  • Each connection maintains an isolated SessionState with its own rPPG buffer — supports concurrent multi-user sessions.
  • Auto-saves measurement records to PostgreSQL when a session exceeds 5 seconds, enabling full history tracking.
  • Frontend hooks (useWebSocket.js + useWebcam.js) handle camera capture, frame encoding, and reconnection logic.

Async Video Processing Pipeline

Background Jobs
  • POST /video/upload-async receives video upload → stores in S3 Object Storage (Supabase) → creates job record in PostgreSQL → dispatches task to Celery + Redis queue.
  • Celery Worker (separate Docker container) downloads from S3 → runs face detection → rPPG inference → signal processing → saves results → deletes S3 file.
  • Job status tracking via GET /video/jobs/{job_id} — Frontend polls every 2 seconds until completion.
  • Full history persistence in PostgreSQL with metadata: HR, HRV (RMSSD/SDNN/PNN50), SNR, age, bandpass params, duration, and timestamps.
  • Dockerized 4-service architecture: Backend, Worker, Redis, Frontend — orchestrated via Docker Compose.

Medical Chatbot — Advanced RAG

NLP / LLM
  • Advanced RAG pipeline: Query Rewriting (LLM-based) → Hybrid Search (FAISS dense + BM25 sparse) → Cross-Encoder Re-ranking → Google Gemini response generation.
  • Knowledge base ingestion from Medical_book.pdf using document loader — supports PDF, Markdown, and plain text formats.
  • FAISS vectorstore with dense embeddings for semantic search, combined with BM25 for keyword-level matching — hybrid retrieval for best accuracy.
  • Lazy-loaded RAG chain — model initializes on first request, keeping cold-start fast and memory efficient.
  • User feedback collection module (feedback_store.py) stores chatbot quality ratings in Supabase for continuous improvement.

Frontend — React + Vite

Web UI
  • 3 main pages: Home (landing with feature guide), Live (real-time webcam + WebSocket vitals), Upload (async video processing + history).
  • Live page: webcam feed with face bounding box overlay, real-time VitalSignCards (HR, SNR, HRV), and interactive BVP waveform chart.
  • Upload page: async file upload with job polling, BVP signal CSV export, age-based bandpass configuration, and inline history viewer.
  • Floating AI chatbot widget accessible from every page — sends queries to backend RAG and displays sourced responses.
  • State management via Zustand, custom hooks for WebSocket/Webcam, and CSS Modules for component-scoped styling.
  • Supabase Auth integration — user login/register to persist and track personal health measurement history over time.

Benchmark Results

Evaluated on 10 users under 3 practical conditions. The system achieves high accuracy (error < 2 bpm) even in noisy environments.
